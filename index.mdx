ThorVG is a high-performance, cross-platform vector graphics engine built in C++. It renders shapes, SVG files, and Lottie animations to a canvas with a clean, minimal API — and fits in a ~150KB binary core.

Used in production by Canva, Godot, LVGL, Tizen, and more.

## Get started

<Steps>
  <Step title="Build and install ThorVG">
    Clone the repository and build with Meson:

    ```bash
    git clone https://github.com/thorvg/thorvg.git
    cd thorvg
    meson setup builddir
    ninja -C builddir install
    ```
  </Step>
  <Step title="Initialize the engine">
    Start the ThorVG engine with the desired number of worker threads:

    ```cpp
    #include <thorvg.h>

    tvg::Initializer::init(4);  // 4 worker threads
    ```
  </Step>
  <Step title="Create a canvas and draw">
    Allocate a pixel buffer, set up a software canvas, and render a shape:

    ```cpp
    static uint32_t buffer[WIDTH * HEIGHT];

    auto canvas = tvg::SwCanvas::gen();
    canvas->target(buffer, WIDTH, WIDTH, HEIGHT, tvg::ColorSpace::ARGB8888);

    auto rect = tvg::Shape::gen();
    rect->appendRect(50, 50, 200, 200, 10, 10);
    rect->fill(0, 120, 200);
    canvas->add(rect);

    canvas->draw();
    canvas->sync();
    ```
  </Step>
  <Step title="Terminate when done">
    Release engine resources when your application exits:

    ```cpp
    tvg::Initializer::term();
    ```
  </Step>
</Steps>

## Rendering backends

ThorVG supports multiple rendering backends — choose based on your target environment:

| Backend | Build option | Best for |
|---------|-------------|----------|
| CPU (Software) | `engines=cpu` | Embedded, server-side, universal |
| OpenGL/ES | `engines=gl` | Mobile, desktop GPU |
| WebGL 2 | `engines=gl` | Web browsers |
| WebGPU | `engines=wg` | Next-gen web & native GPU |

<Tip>
  The CPU backend is enabled by default and requires no external dependencies. Start here for maximum portability.
</Tip>
